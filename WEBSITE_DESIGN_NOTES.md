# Website Design Notes

The website should behave more like a serious mathematical tutorial than a marketing page or dashboard.

Reference:

- https://invariances.org/ginvariance-tutorial/

Useful design lessons from that tutorial:

- Start with a preface that states the method and scope.
- Keep a visible table of contents.
- Prefer long-form mathematical flow over small disconnected cards.
- Use anchored sections so the learner can return to definitions, examples, and exercises.
- Introduce abstract objects through examples and "how can we construct this?" transitions.
- Let the page feel like a course reader, not a product page.

For this course, the website should support:

- Reading the course path
- Opening full Markdown lectures
- Previewing base cases
- Tracking progress
- Writing lecture worksheets

The source slide decks remain the pedagogy reference. The invariances tutorial is mainly a website-format reference.
