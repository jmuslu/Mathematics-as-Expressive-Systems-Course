# Question Bank

This folder is the reservoir for candidate problems, reserve exercises, source-inspired variants, and pilot trails before they are promoted into the lecture modules.

The lecture modules should stay selective. A module trail should contain the strongest problems for the learning arc:

```text
base case -> by-hand manipulation -> answer check -> intuition -> modeling implication
```

Reserve exercises can be broader, harder, more technical, or more source-adjacent, as long as provenance is tracked.

Use [Problem Trail Style](../PROBLEM_TRAIL_STYLE.md) for analogy rules, bank roles, and promotion criteria.

## Files

- `module-06-tensors-typed-relations-pilot.md` - first reviewed pilot using the dinner-table analogy.
- `module-07-covariance-contravariance-bank.md` - reserve and lecture-candidate problems for variance, metrics, slot grammar, and wedge checks.
- `module-08-hermitian-structure-bank.md` - reserve and lecture-candidate problems for complex inner products, unitary maps, adjoints, and Hermitian forms.
- `module-13-16-symmetry-bank.md` - reserve and lecture-candidate problems for groups, actions, representations, equivariance, and Reynolds averaging.
- `problem-template.md` - metadata template for future bank entries.

## Promotion Rule

Before a problem moves from the bank into a module, check:

1. The problem has one clear mathematical point.
2. The arithmetic or symbolic answer check has been verified.
3. The problem supports the module's current lecture promise.
4. The source status is original, public/open, or adapted according to `QUESTION_BANK_SOURCE_POLICY.md`.
5. The wording fits the course style: concrete first, abstraction second, failure mode last.

## Reserve Rule

Keep a problem in reserve when it is correct but:

- too long for the lecture trail,
- too advanced for the module's first pass,
- redundant with a cleaner problem,
- source-adjacent and not yet fully rewritten,
- useful as an optional challenge.
