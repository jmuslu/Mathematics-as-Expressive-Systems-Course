A Complete Beginner's Guide to

Mathematics as Expressive Systems

Applied math for representing friendship, dinner plans, debate boards, uncertainty, symmetry, local consistency, and evolving graph structure.

Preface

  • This is not a RAG tutorial. External memory is one application; the course is about expressive mathematical systems.
  • The guiding object is a structured representation: typed states, relations, transformations, invariants, uncertainty, decay, and reinforcement.
  • Each module now includes a hand problem trail: concrete setup, calculation, answer check, and design interpretation.
  • Important mature topics such as Hermitian structure, group actions, G-invariance, representation theory, sheaves, and categorical databases are first-class modules.

Method and Further Readings

  • For applied category theory: Seven Sketches in Compositionality.
  • For tensor maturity: dual spaces, Hermitian forms, active/passive transformations, tensor products, antisymmetric tensors, and representation tensor products.
  • For group intuition: Visual Group Theory, then representation theory and G-invariance.
  • For graph systems: Easley and Kleinberg, Diestel, algebraic graph theory, hypergraphs, and spectral graph theory.
  • For local-to-global validation: sheaves, cosheaves, graphical models, and message passing.

Preview: What Counts as a Safe Transformation?

Suppose a dinner seating graph is relabeled. The names of the seats change, but the structure should not. This is a group action. If a feature changes under relabeling, it has learned an artifact of storage, not a property of the graph. 

A' = P A P^T

A good graph-level summary should be invariant to this transformation. A node-level embedding should usually be equivariant: when nodes are relabeled, the embeddings relabel with them.

Recurring Product Questions

A major thread is learning which operation is legal and what it means. Inner product, outer product, matrix product, tensor product, Kronecker product, wedge product, contraction, categorical product, and operadic composition are not interchangeable ways to combine things. 

inner product: alignment
outer product: generated relation
tensor product: joint typed state
wedge product: oriented antisymmetric content
contraction: legal slot pairing
composition: process after process
gluing: local data into global consistency
