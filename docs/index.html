A Complete Beginner's Guide to

Mathematics as Expressive Systems

Applied math for dynamic epistemic graphs, G-invariance, category theory, sheaves, topology, and LLM second-brain architecture.

Preface

  • This course is not a RAG tutorial. It is a mathematical path toward designing external cognitive architecture.
  • The guiding object is a dynamic epistemic graph: typed memories, evidence, transformations, validations, decay, and reinforcement.
  • The course follows the original ML math slide style: base case first, abstraction second, design implication last.
  • Important mature topics such as Hermitian structure, group actions, G-invariance, representation theory, sheaves, and categorical databases are first-class modules.

Method and Further Readings

  • The exposition style follows the supplied lecture PDFs: prerequisites, concrete problem, operation, derivation, failure mode, practice.
  • For applied category theory: Seven Sketches in Compositionality.
  • For tensor maturity: Jeevanjee's An Introduction to Tensors and Group Theory for Physicists, especially dual spaces, Hermitian forms, active/passive transformations, tensor products, antisymmetric tensors, and representation tensor products.
  • For group intuition: Visual Group Theory, then representation theory and G-invariance.
  • For graph systems: Easley and Kleinberg, Diestel, algebraic graph theory, hypergraphs, and spectral graph theory.
  • For local-to-global validation: sheaves, cosheaves, graphical models, and message passing.

Preview: What Counts as a Safe Transformation?

Suppose a memory graph is relabeled. The names of the nodes change, but the meaning should not. This is a group action. If a feature changes under relabeling, it has learned an artifact of storage, not a property of the graph. 

A' = P A P^T

A good graph-level memory summary should be invariant to this transformation. A node-level memory embedding should usually be equivariant: when nodes are relabeled, the embeddings relabel with them.

Recurring Product Questions

A major thread inherited from the original coursework is learning which operation is legal and what it means. Inner product, outer product, matrix product, tensor product, Kronecker product, wedge product, contraction, categorical product, and operadic composition are not interchangeable ways to combine things. 

inner product: alignment
outer product: generated relation
tensor product: joint typed state
wedge product: oriented antisymmetric content
contraction: legal slot pairing
composition: process after process
gluing: local data into global consistency
