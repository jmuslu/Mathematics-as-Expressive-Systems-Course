# Question Bank Source Policy

This course is noncommercial, but noncommercial use and an MIT license do not automatically clear third-party exercises, papers, slide decks, or textbook content.

The working rule is:

```text
use sources for orientation, then write original course problems unless the source license clearly allows reuse
```

## Source Classes

| Source type | Use in question bank |
| --- | --- |
| Original course problems | May be included directly |
| Public-domain or permissively licensed material | May be quoted or adapted with attribution and license notes |
| Creative Commons material | Follow the exact CC terms, including attribution, noncommercial, no-derivatives, or share-alike limits |
| Institutional-login material | Use for topic orientation and verification; rewrite as original problems unless explicit reuse rights are clear |
| Copyrighted textbooks or papers | Use for mathematical orientation; avoid copying problem text unless permission or license allows it |

## Problem Metadata

Each bank problem should carry enough information to audit later:

```text
Topic:
Role: warm-up / computation / derivation / conceptual check / counterexample / reserve
Status: lecture candidate / reserve
Source use: original / inspired by / adapted from / quoted from
Source note:
Answer check:
```

## Selection Rule

The central question bank can be broad. A lecture should only receive the problems that directly support its learning arc:

```text
base case -> by-hand manipulation -> answer check -> intuition -> modeling implication
```

Reserve exercises should stay in the bank when they are useful but distract from the main lecture.

## Verification Rule

Every promoted lecture problem needs a short answer check. For more abstract topics, include at least one of:

- a tiny numerical example,
- a symbolic identity check,
- a counterexample,
- a plain-language interpretation of what failed or survived.

This keeps the problem bank useful even when source material is abstract or notation-heavy.
